## §4 Findings

---

### R9-001 · DEFECT · `forge-stamp-version.sh` destroys `load_schema_version()` on every stamp run

**Pattern:** #8 Silent Degradation, #22 Dual Authority, #10 Happy Path Only
**Files:** `scripts/forge-stamp-version.sh` (lines 57–63), `proxmox-bootstrap/version.py`, `proxmox-bootstrap/migration_manager.py:85`, `proxmox-bootstrap/bootstrap_state.py:58`, `proxmox-bootstrap/package_verifier.py:181`

**Description:**
`forge-stamp-version.sh` overwrites `version.py` via heredoc containing ONLY the `SCHEMA_VERSION` constant — no `load_schema_version()` function:

```bash
cat > "$VERSION_PY" <<EOF
SCHEMA_VERSION: str = "${VERSION}"
EOF
```

After stamping, all three callers call `mod.load_schema_version()` which raises `AttributeError`, is caught by `except Exception`, emits a `warnings.warn()` to stderr (which may go unnoticed), and returns the hardcoded fallback `"2026-06-09_00-00-00_0000000"`. The newly stamped version is silently discarded. `CURRENT_SCHEMA_VERSION` in all three modules remains the baseline forever regardless of what `forge-stamp-version.sh` produces.

This is a stamp ceremony that appears to succeed but silently fails to update the version constant in running code.

**Fix applied:** Removed `load_schema_version()` from `version.py` (making it a pure constants file, matching what the stamp script already produces). Updated all three callers to access `str(mod.SCHEMA_VERSION)` directly.

---

### R9-002 · RISK · `verify()` exit code 2 and `verify_state_descriptor()` not-found path are untested

**Pattern:** #31 Regression Blind Spot
**File:** `tests/unit/test_package_verifier.py`

**Description:**
R8-003's minimum test list explicitly required "verify() returns 2 when descriptor is absent". The added `test_package_verifier.py` covers exit codes 0 and 1 but not exit code 2. Similarly, the `verify_state_descriptor()` not-found path (which callers map to exit code 2) is also untested.

Exit code 2 is the critical signal distinguishing "package never stamped" from "hash mismatch" — if this branch regresses, `forge-migrate.sh` would treat an uninitialized deployment as a hash mismatch and emit confusing error messages.

**Fix applied:** Added `test_verify_missing_descriptor` and `test_verify_state_missing_descriptor` to `test_package_verifier.py`.
